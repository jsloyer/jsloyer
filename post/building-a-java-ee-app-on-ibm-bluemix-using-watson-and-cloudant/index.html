Today we are going to be building an example app using Java, Cloudant, and Watson.

Ok let‚Äôs talk through what this app is going to do before we build it.

Meet Ivy (hello!)

She‚Äôs a talent manager at a growing tech startup.

She‚Äôs looking for a new hire that would be a good fit on her team but the company is so popular that she has a huge inventory of resumes to sort through. She‚Äôs looking for tools to help her, and tools beyond just simple tag filters.

With Watson services she can also start to solve for a problem like, “I‚Äôm looking for another developer like “[insert cool employee].”

So in this case, the application can issue queries such as,

Find me a Developer like Craig Smith.Then search through all possible candidate and return a ranked list of candidates sorted by highest-to-lowest percentage of personality resemblance. From here, searches can be refined by including technical skills. Find me a Developer like Craig Smith, and knows Java, C and Python.

– Ivy (HR Manager)

Create the application

First we need to sign into Bluemix, so open your browser and head on over to Bluemix. We need to login. ¬†Up at the top lets go ahead and click on Catalog.

On the right hand side we need to give our app a name. ¬†Please note that this name must be unique. ¬†Also remember this name as we will need it later‚Ä¶

Click create.

Bluemix will start deploying our app and Bluemix will start our app.

So next we need to add Cloudant and Watson to our app!

So do you that make sure you are in the dashboard and have your app open like the screencap below. ¬†We will then click the “Add A Service” button.

First let‚Äôs add Watson. ¬†Scroll down until you find the “User Modeling” Service. ¬†Go ahead and click that.

Again, all we have to do is click “Create” on the right hand side.

Bluemix is going to ask us if we want to restage our app, we should click the “OK” button. ¬†What Bluemix is asking us here is since we made a change to our app, the app needs to be restarted to bind in Watson.

Next,¬†let‚Äôs click “Add A Service” again.

This time scroll down to the bottom and choose “Cloudant”.

Then click “Create” on the right hand side.

It is going to ask us to restage our app again, go ahead and click “OK”.

Now we have all our services added. ¬†Let‚Äôs go ahead and start getting some data imported.

On the dashboard for the app let‚Äôs go ahead and click on Cloudant.

Explanation of Cloudant

So let‚Äôs pause and explain what Cloudant is. ¬†Cloudant is a No-SQL database that is based on CouchDB. ¬†The big difference between a relationship database and No-SQL database is illustrated below. ¬†In the relationship database you have to have multiple tables to represent the data and have to use SQL and JOIN statements to get data from both tables. ¬†In a No-SQL database which is commonly referred to as a document store database you stores JSON documents/data (as illustrated on the left below).

  public CloudantClient()

    this.httpClient = null;

     try {
        String VCAP_SERVICES = System.getenv("VCAP_SERVICES");
        JSONObject vcap;
        vcap = (JSONObject) JSONObject.parse(VCAP_SERVICES);

        cloudant = (JSONArray) vcap.get("cloudantNoSQLDB");

        e.printStackTrace();

      this.port = Config.CLOUDANT_PORT;
      this.host = (String) cloudantCredentials.get("host");
      this.username = (String) cloudantCredentials.get("username");
      this.password = (String) cloudantCredentials.get("password");
      this.name = Config.CLOUDANT_NAME;
      this.dbc = this.createDBConnector();
  }

In the bottom it should say “BUILD SUCESSFULL”.

The next and last step involves deploy your app to bluemix. ¬†This step requires the Cloud Foundry CLI to be installed (if you haven‚Äôt done this yet scroll back up the pre-req‚Äôs section above).

If you are on Windows¬†open up the command prompt, if you are on a Mac or Linux open up the terminal.

Type the following.
